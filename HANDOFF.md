# HANDOFF.md — Order Management (Tier 4: Senior Associate)

## What was built

A complete, production-grade Order Management application spanning all four tiers of the Excalibur assignment. Every file was written from scratch within this session; the scaffolded `.gitkeep` placeholders have been replaced with real code.

## Phases delivered

### Phase 1 — Tier 1 (Junior)
- **Alembic migration** `backend/app/db/migrations/versions/0001_initial.py` — creates `order_date`, `order_detail`, `order_combined` with correct types (`UUID PK`, `DATE`, `NUMERIC(12,2)`, `VARCHAR(500)`), FK cascade, and indexes.
- **Seed data** `db/seed.sql` — 30 rows spanning Jan–Dec 2025, amounts from $7 to $3,200 (covers <20, >300, and between).
- **`docker-compose.yml`** — Postgres 16, Adminer, backend (hot-reload), frontend (Vite dev), report-runner (profile=report).
- **FastAPI app** — `POST /orders/combine` + `GET /healthz` + `GET /readyz`.
- **Generic sort** `backend/app/services/sort.py` — `sort_by(items, key_fn, reverse)` with `SupportsRichComparison` Protocol, used at both call sites.
- **Transactional combine** — TRUNCATE + bulk INSERT in one transaction with rollback on exception.
- **React frontend** — filter form + results table in `src/features/combine/Combine.tsx`.

### Phase 2 — Tier 2 (Associate)
- **Full REST CRUD** `GET/POST/PUT/DELETE /orders/{id}` in `backend/app/api/rest/orders.py`.
- **CSV export** `GET /orders/export.csv` (same filter params as combine).
- **React features** — orders list, create form, edit form, **Tab/Card toggle** (Table view = blue/white grid; Card view = amber/orange gradient tiles — visibly distinct layout, spacing, color emphasis).
- **Frontend Dockerfile** — multi-stage: dev (Vite), runtime (nginx).
- **GitHub Actions `ci.yml`** — lint, backend tests with coverage artifact, frontend tests, build check, Terraform module validation.
- **Backend coverage** — unit + integration tests targeting ≥60% line coverage on `app/`.

### Phase 3 — Tier 3 (Mid-Level)
- **All 12 Terraform modules** under `infra/modules/`: `network`, `kms`, `rds`, `ecs-cluster`, `ecs-service`, `alb`, `cognito`, `s3-reports`, `eventbridge-schedule`, `ecr`, `observability`, `github-oidc`.
- **OpenAPI** auto-generated by FastAPI; Swagger at `/docs`, Redoc at `/redoc`; disabled in prod via `ENABLE_DOCS=false`.
- **Strict DTO separation** — ORM models never cross the API boundary; all endpoints return Pydantic schemas.
- **Ansible** — `app-deploy`, `db-migrate`, `healthcheck` roles; `deploy.yml` and `rollback.yml` playbooks; idempotent (re-run = changed=0 when image tag unchanged).
- **JasperReports** — `orders_by_month.jrxml` with `P_DATE_FROM`, `P_DATE_TO`, `P_AMOUNT_MIN`; `report-runner/runner.py` connects to RDS, runs via `jasperstarter`, uploads PDF to S3. Local: `make report`.
- **React Router v6** — 5 lazy-loaded routes (Index/Orders, /orders/new, /orders/:id, /combine, /reports).
- **Frontend tests** — Vitest + React Testing Library in `frontend/tests/` targeting ≥60% coverage.
- **Architecture diagram** — `docs/architecture.svg` (SVG rendered from code) and `docs/architecture.png` (rsvg-convert).
- **CD pipeline** — `cd.yml`: single workflow; env input defaults to dev; push to main→dev, tag v*.*.*→qa, workflow_dispatch→any env. Jobs use environment-scoped secrets..

### Phase 4 — Tier 4 (Senior Associate)
- **Terragrunt layering** — `infra/live/root.hcl` DRYs provider + S3+DDB backend; `_envcommon/*.hcl` holds per-module common inputs; `dev/qa/prod` each have their own `env.hcl` and per-module `terragrunt.hcl` files. All 11 modules covered across all 3 environments.
- **RDS Multi-AZ** — `rds_multi_az=true` in qa/prod; KMS CMK encryption; `rds.force_ssl=1` parameter group; credentials only in Secrets Manager; ECS task reads via `secrets` block — no password in any env var, tfvars, or repo file.
- **GraphQL** — Strawberry schema at `/graphql`: `Query.joinedOrders`, `Mutation.upsertOrder`, `Subscription.orderCombinedRegenerated` (polling fallback). `selectinload` on `OrderDate.detail` prevents N+1; asserted in `test_combine_no_n_plus_1`.
- **Cognito OIDC** — `viewer` and `editor` groups; server-side enforcement via FastAPI dependencies (`require_viewer`, `require_editor`); client-side via `useAuth().isEditor` guard hiding mutate buttons and protecting routes; local dev auto-bypasses when `COGNITO_USER_POOL_ID` is unset.
- **Ansible Vault** — per-env `group_vars/{dev,qa,prod}/vault.yml` (plaintext stubs — encrypt with `ansible-vault encrypt` before committing). `ansible.cfg` references `~/.vault_pass`.
- **Multi-env promotion CI/CD** — single `cd.yml` with an `environment` input (default: `dev`). Push to `main` → dev; tag `v*.*.*` → qa; `workflow_dispatch` with any env. Each GitHub Environment (`dev`/`qa`/`prod`) owns its own scoped secrets (`AWS_DEPLOY_ROLE_ARN`, `ALB_URL`, `COGNITO_AUTHORITY`, etc.); all jobs reference `secrets.AWS_DEPLOY_ROLE_ARN` uniformly. `prod` protection rules (required reviewers) gate the `tf-apply` job.
- **Playwright e2e** — `frontend/e2e/orders.spec.ts` covers: auth setup, viewer checks, create → list → combine → CSV export → reports page.
- **Observability** — structured JSON logging via structlog with OTel trace_id correlation; ADOT collector sidecar in task definition; spans → X-Ray; metrics/logs → CloudWatch; `observability/dashboards/orders-overview.json` (request rate, p95 latency, 5xx rate, ECS CPU/memory, unhealthy hosts); alarms: 5xx > 2% for 5min, p95 > 1s for 5min, unhealthy hosts > 0 for 2min.
- **Scheduled reports** — EventBridge schedule `cron(0 6 * * ? *)` → ECS RunTask of `report-runner` → PDF to `s3://orders-reports-{account}-{region}-{env}/{YYYY-MM-DD}.pdf`; `/reports` route fetches presigned URL.
- **Runbooks** — `failed-deploy.md`, `db-credential-rotation.md`, `healthcheck-rollback.md` in `docs/runbooks/`.

## Before you deploy to AWS

1. **Replace placeholders** in env.hcl files:
   - `REPLACE_WITH_DEV_AWS_ACCOUNT_ID` → your AWS account ID
   - `REPLACE_ORG/REPLACE_REPO` in github-oidc configs → your GitHub repo
   - `REPLACE_ACCOUNT/REPLACE_CERT_ID` in ALB configs → real ACM cert ARN

2. **Bootstrap Terraform state** (first apply only):
   ```bash
   aws s3api create-bucket \
     --bucket order-management-tfstate-{account_id}-us-east-1 \
     --region us-east-1

   aws s3api put-bucket-versioning \
     --bucket order-management-tfstate-{account_id}-us-east-1 \
     --versioning-configuration Status=Enabled

   aws dynamodb create-table --table-name order-management-tfstate-locks \
     --attribute-definitions AttributeName=LockID,AttributeType=S \
     --key-schema AttributeName=LockID,KeyType=HASH \
     --billing-mode PAY_PER_REQUEST \
     --region us-east-1
   ```

3. **Encrypt Ansible Vault files** (contains ARN references, not actual secrets):
   ```bash
   ansible-vault encrypt ansible/group_vars/dev/vault.yml
   echo "your-vault-password" > ~/.vault_pass
   ```

4. **Set GitHub repository secrets** (see `.github/workflows/cd.yml`):
   - `AWS_DEPLOY_ROLE_ARN_DEV`, `AWS_DEPLOY_ROLE_ARN_QA`, `AWS_DEPLOY_ROLE_ARN_PROD`
   - `DEV_ALB_URL`, `DEV_COGNITO_AUTHORITY`, `DEV_COGNITO_CLIENT_ID`
   - `E2E_VIEWER_EMAIL`, `E2E_VIEWER_PASSWORD`

5. **Run Terragrunt** (creates all AWS resources):
   ```bash
   cd infra/live/dev
   terragrunt run-all apply
   ```
